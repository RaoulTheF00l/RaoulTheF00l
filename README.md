# Hey, I'm Raoul

I'm a solo game developer focused on **Godot**, **GDScript**, **JavaScript automation**, 2D prototypes, RPG systems, pixel-style UI, and small playable gameplay loops.

I like building projects from the ground up: player controllers, scene transitions, collision setup, UI panels, resource systems, turn-based combat, idle-game economies, and small tools that make a project easier to understand and expand.

## What I’m building toward

My current focus is becoming a stronger gameplay programmer by finishing small, understandable prototypes instead of over-scoping huge dream games too early.

Right now, I am practicing:

* Godot 4 project structure
* 2D movement and collision
* UI layout and menu flow
* RPG interaction systems
* turn-based combat foundations
* idle / incremental game loops
* JavaScript automation scripts
* clean file organization and readable code

## Featured projects

### [Tower-Slice](https://github.com/RaoulTheF00l/Tower-Slice)

A small Godot tower-progression prototype built around one complete playable loop: start in a hub, enter a tower floor, fight an enemy, gather materials, return to town, craft a weapon, and complete the demo.

**Built with:** Godot, GDScript
**Focus:** playable vertical slice, scene flow, crafting loop, hub structure, small-scope game design

This is currently one of my strongest portfolio projects because it is scoped around a finishable gameplay slice instead of an oversized RPG plan.

### [adventureland_scripts](https://github.com/RaoulTheF00l/adventureland_scripts)

A modular JavaScript automation project for Adventure Land. The scripts are organized around party roles, shared behavior modules, combat routing, movement, inventory handling, town runs, item progression, and an in-game status panel.

**Built with:** JavaScript
**Focus:** modular scripting, role-based behavior, automation, party coordination, runtime status tracking

Current systems include:

* party runner loop
* role modules for mage, priest, ranger, and merchant
* movement and combat modules
* inventory and town-run logic
* gold/hr and XP/hr status HUD
* merchant support for restocking, loot transfer, and party assistance

### [Persona: Out of Phase](https://github.com/RaoulTheF00l/P5OOP)

A non-commercial Godot 4 fan demake based on my own Persona-inspired fan fiction. This project is being used to practice RPG-style systems, daily-life structure, turn-based combat, player stats, scene organization, and top-down exploration.

**Built with:** Godot 4, GDScript
**Focus:** RPG systems, combat logic, social stats, time management, UI flow, scene management

Current systems include:

* title screen and hub flow
* player movement
* social stat tracking
* time-slot management
* basic turn-based combat
* battle rewards
* placeholder Leblanc exploration
* NPC and enemy interaction planning

### [Hero of the Grove](https://github.com/RaoulTheF00l/Hero-of-The-Grove)

A small top-down action-adventure prototype built in Godot. The goal is to create a focused vertical slice with a forest clearing, NPC interaction, a basic weapon, a tiny dungeon, one enemy, one puzzle, one chest, and a demo-complete room.

**Built with:** Godot, GDScript
**Focus:** top-down movement, tilemap workflow, interaction systems, dungeon structure, small adventure-game design

This project is currently in early prototype setup and is being used to study classic 2D adventure structure in a manageable scope.

---

### [Chronic Clicker](https://github.com/RaoulTheF00l/ChronicClicker)

An early idle-incremental prototype built in Godot. The core loop focuses on harvesting product, selling it for cash, hiring workers, hiring farmers, and increasing production through simple automation.

**Built with:** Godot, GDScript
**Focus:** idle-game economy, UI labels/buttons, resource tracking, automation, bug fixing

This project helped me practice basic Godot fundamentals like scenes, signals, buttons, labels, economy variables, and small gameplay loops.
